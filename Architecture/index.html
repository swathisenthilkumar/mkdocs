nan 1.2.2 2 306 nan Verify that communications between application components, including APIs, middleware and data layers, are authenticated. Components should have the least necessary privileges needed. (C3) nan nan nan nan

Access Control Architectural Requirements 1.4.1 2 602 nan Verify that trusted enforcement points such as at access control gateways, servers, and serverless functions enforce access controls. Never enforce access controls on the client. nan nan nan nan
nan 1.4.2 2 284 nan Verify that the chosen access control solution is flexible enough to meet the application's needs. nan nan nan nan
nan 1.4.3 2 272 nan Verify enforcement of the principle of least privilege in functions, data files, URLs, controllers, services, and other resources. This implies protection against spoofing and elevation of privilege. nan nan nan nan
nan 1.4.4 2 284 nan Verify the application uses a single and well-vetted access control mechanism for accessing protected data and resources. All requests must pass through this single mechanism to avoid copy and paste or insecure alternative paths. (C7) nan nan nan nan
nan 1.4.5 2 275 nan Verify that attribute or feature-based access control is used whereby the code checks the user's authorization for a feature/data item rather than just their role. Permissions should still be allocated using roles. (C7) nan nan nan nan
